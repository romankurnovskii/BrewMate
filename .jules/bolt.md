## 2024-06-25 - Caching computed properties on data objects to optimize massive array filtering
**Learning:** In a codebase dealing with large un-paginated datasets (like Homebrew's ~100k apps), creating inline closures and repeating string transformations (like `.toLowerCase()`) inside `.filter()` loops during keystroke search causes severe main-thread blocking and UI stutter. The bottleneck isn't the iteration itself, but the repeated object property lookups and string allocations.
**Action:** Always move invariant string transformations outside the loop. Use the data objects themselves to cache the results of expensive operations (like calculated categories or lowercased strings) by adding optional private properties (e.g., `_category`, `_nameLower`). Order filter conditions by execution speed (direct boolean checks > map lookups > string includes) and use early returns to exit fast before running expensive string comparisons.
