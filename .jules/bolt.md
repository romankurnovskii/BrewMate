## 2024-04-18 - Optimize Large Array Filtering in UI Thread
**Learning:** When filtering a very large dataset (~100k items) for a "live search" feature, running string `.toLowerCase()` conversions and creating inline closures inside a `.filter()` array method on every keystroke causes significant blocking of the main UI thread, resulting in a sluggish user experience. Standard optimizations like debouncing only mask the underlying computational inefficiency of repeatedly re-evaluating unchanged data.
**Action:** Move invariant computations (like lowercasing the search term) outside the loop. More importantly, memoize expensive per-item computed properties (like lowercased strings and calculated categories) directly on the item objects during their first pass. Use early returns based on condition speed (e.g., direct equality checks before string inclusion checks) to short-circuit the filter logic as early as possible.
