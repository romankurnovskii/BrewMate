## 2024-05-14 - Move invariant transformations out of filter loops
**Learning:** Found a major performance bottleneck where invariant string transformations like `searchTerm.toLowerCase()` were being re-computed inside the $O(N)$ `allApps.filter()` loop in `src/renderer/renderer.ts`. Given the collection holds 13,000+ items, computing this $13000$ times per UI input event is massively inefficient and negatively impacts responsiveness. Additionally, discovered that applying a `|| ''` defensive check against the search term ensures we don't crash when running transformations on empty values (which is a codebase constraint). Finally, I learned that modifying package.json or applying global formatting (`npx prettier --write "src/**/*.{ts,tsx}"`) creates unacceptable noise in the PR.
**Action:** When working with large collections (like `allApps`), ensure any invariant transformations, especially string casing, are lifted out of the loop and executed exactly once. Always respect the 'Never modify package.json' boundary. Use `git restore` to revert changes outside the immediate scope of the feature to keep the PR clean.
