## 2026-04-06 - Stream and Network Optimization for Large Datasets
**Learning:** Fetching extremely large API payloads (like the ~30MB Homebrew JSONs) without compression wastes network bandwidth and significantly increases load time. Additionally, accumulating streamed chunks via string concatenation (`data += chunk`) forces V8 to repeatedly allocate memory for larger strings (an $O(N^2)$ operation) and risks corrupting multi-byte characters when split across chunks.
**Action:** Always send `Accept-Encoding: gzip, deflate, br` headers when fetching large JSON responses and decompress them dynamically with `zlib`. Accumulate chunks into an array and use `Buffer.concat(chunks).toString('utf8')` at the end to efficiently reconstruct the payload in an $O(N)$ operation without character corruption.
