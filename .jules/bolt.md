## 2024-04-28 - Optimization Pattern for Array Processing
**Learning:** For large dataset iterations (like ~100k `allApps`), array methods combining inline closures/IIFEs inside `.filter()` operations can block the UI thread. Short-circuit evaluations correctly order condition checks (type > search > category) effectively removing items via fast property checks before invoking expensive classification routines. Furthermore, reusing invariant strings computed (like `.toLowerCase()`) directly via optional caching fields on the items vastly reduces redundant string allocations.
**Action:** Always move invariant properties out of large loops, prefer explicit early-return logical branches (or ordered short-circuit operations) to avoid slow deep evaluations where possible, and securely compute/cache expensive derivation fields explicitly onto iteration elements dynamically.
