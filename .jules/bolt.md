## 2024-03-29 - [Optimization of large array filter in React/Electron renderer]
**Learning:** In applications dealing with large datasets (~100k items, like Homebrew package data), putting inline closures (IIFEs), calculating string transformations (like `.toLowerCase()`), or computing complex properties via string manipulation inside `Array.prototype.filter()` heavily bottlenecks the UI thread during frequent updates like typing/searching.
**Action:** Move all invariant transformations (e.g., `searchTerm.toLowerCase()`) completely out of the `.filter()` loop. Re-order conditionals by speed (direct property comparisons -> set `has` checks -> expensive string manipulation) to enable early returns. Memoize computed results (like item category) directly onto the object the first time they are processed, avoiding recalculation during future key presses.
