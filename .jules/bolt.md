## 2024-05-02 - Frontend Searching Optimization
**Learning:** For large datasets like Homebrew's application list (~100k records), inline property formatting such as `(app.description || '').toLowerCase()` within the iteration block (like a filter callback or mapping function) presents a significant performance overhead that blocks the main UI thread during keystrokes in the search bar. This translates into heavy frame drops and noticeable search latency.
**Action:** When filtering or mapping very large lists on the frontend, precompute and memoize computed, format-heavy attributes (like dynamically generated strings for lowercase search parsing and classification categories) directly onto the object items as cached private properties (`_propertyLower`, `_category`) upon data load, thereby converting repeated inline generation during the search loop into quick constant-time field accesses.
