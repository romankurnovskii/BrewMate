## 2024-06-25 - Renderer Performance Optimization
**Learning:** For extremely large arrays (~100k items) operated on high-frequency UI events like keypress search filters, recalculating invariant properties inside the `.filter()` loop causes major main thread blocking and noticeable jitter. Repeatedly calling `.toLowerCase()` on string fields for every item creates enormous garbage collection overhead from temporary string allocations.
**Action:** When filtering massive datasets on the frontend, pre-compute invariant fields (like lowercase variants of fields you search against, or calculated categories) and memoize them directly on the object instance (e.g., `app._nameLower ??= app.name.toLowerCase()`). Couple this with early returns to short-circuit the filter logic as fast as possible.
