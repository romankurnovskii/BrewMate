## 2024-04-19 - Optimizing Array Filter for Large Datasets
**Learning:** For large datasets (like the app's ~100k `allApps`), avoiding inline closures/IIFEs inside `.filter()` loops and moving invariant transformations outside the loop is critical. Using early returns ordered by condition speed (direct property checks > set lookups > string includes) and caching expensive computed properties (like calculated categories and toLowerCase results) directly onto the item objects prevents UI thread blocking during keystroke searches.
**Action:** Always extract invariant transforms (e.g. search string lowercasing) outside the loop for array filtering. Use cached properties (`_category`, `_nameLower`, etc.) directly on items when executing multiple iterations of filters on a large dataset.
