## 2024-04-27 - Optimizing Array.prototype.filter() on large datasets
**Learning:** When filtering large datasets (~100k records, like Homebrew's combined formula/cask lists) during user interaction (like keystroke searches), invariant computations inside the `.filter()` callback (e.g., calling `.toLowerCase()` on the search term) and repeated string transformations on the items create significant overhead that can block the UI thread.
**Action:** For performance-critical `.filter()` loops, move invariant logic (like search term normalization) outside the loop. Sequence filter conditions by speed: direct property checks > Set lookups > string inclusions. For string inclusions, memoize/cache the transformed properties (e.g., lowercased names and descriptions) directly on the item objects during their first evaluation to avoid recomputing them on every subsequent keystroke.
