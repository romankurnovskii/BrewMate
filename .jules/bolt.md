## 2024-05-24 - Optimizing Array Filtering over Large Data Sets
**Learning:** In the Homebrew GUI context, the `App` array data size can reach upwards of ~100k items. Standard functional iteration patterns using `.filter()` with inline closures/IIFEs, and repeated invariant string transformations (e.g. `searchTerm.toLowerCase()` or recalculating `getCategoryForApp()`) severely impact UI thread performance on keystroke searches due to massive redundant allocations and slow string combinations.
**Action:** Always move invariant properties outside loops (like `search.toLowerCase()`), employ fast-path early returns (Type check > Set lookup for Installed > Computed categories > Substring matching), and lazy-initialize caching properties directly on the objects (e.g., `_category`, `_nameLower`) rather than re-computing them inline.
